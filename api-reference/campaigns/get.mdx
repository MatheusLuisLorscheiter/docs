---
title: "Consultar Campanha"
api: "GET /api/campaigns/{id}"
description: "Consulta os detalhes de uma campanha específica"
---

<Tip>
Retorna os detalhes completos de uma campanha específica, incluindo status atual e progresso.
</Tip>

<Warning>
Você precisa de um access token válido no header X-Access-Token para usar este endpoint.
</Warning>

## Path Parameters

<ParamField path="id" type="number" required>
  ID único da campanha que deseja consultar
</ParamField>

## Headers

<ParamField header="X-Access-Token" type="string" required>
  Token de acesso obtido no registro
</ParamField>

<RequestExample>

```bash cURL
curl -X GET https://{url-key}/api/campaigns/{campaign_id} \
  -H "X-Access-Token: {access_token}"
```

```javascript JavaScript
const campaignId = {campaign_id};
const response = await fetch(`https://{url-key}/api/campaigns/${campaignId}`, {
  method: 'GET',
  headers: {
    'X-Access-Token': '{access_token}'
  }
});

const campaign = await response.json();
console.log(campaign);
```

```python Python
import requests

campaign_id = {campaign_id}
headers = {
    'X-Access-Token': '{access_token}'
}

response = requests.get(
    f'https://{url-key}/api/campaigns/{campaign_id}',
    headers=headers
)

campaign = response.json()
print(campaign)
```

```php PHP
<?php
$campaign_id = {campaign_id};
$headers = [
    'X-Access-Token: {access_token}'
];

$curl = curl_init();
curl_setopt_array($curl, [
    CURLOPT_URL => "https://{url-key}/api/campaigns/{$campaign_id}",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => $headers
]);

$response = curl_exec($curl);
curl_close($curl);

$campaign = json_decode($response, true);
print_r($campaign);
?>
```

</RequestExample>

<ResponseExample>

```json Sucesso (200 OK)
{
  "id": 1,
  "name": "{campaign_name}",
  "status": "ACTIVE",
  "totalContacts": 2,
  "sentMessages": 2,
  "failedMessages": 0,
  "createdAt": "{creation_datetime}",
  "updatedAt": "{update_datetime}",
  "scheduledFor": null,
  "evolutionUrl": "{evolution_url}",
  "evolutionInstance": "{evolution_instance}",
  "message": null
}
```

```json Erro (400 Bad Request)
{
  "id": null,
  "name": null,
  "status": null,
  "totalContacts": 0,
  "sentMessages": 0,
  "failedMessages": 0,
  "message": "{error_message}"
}
```

</ResponseExample>

## Response

<ResponseField name="id" type="number">
  ID único da campanha
</ResponseField>

<ResponseField name="name" type="string">
  Nome da campanha
</ResponseField>

<ResponseField name="status" type="string">
  Status atual: DRAFT, SCHEDULED, RUNNING, PAUSED, COMPLETED, CANCELLED
</ResponseField>

<ResponseField name="totalContacts" type="number">
  Total de contatos na campanha
</ResponseField>

<ResponseField name="sentMessages" type="number">
  Mensagens enviadas com sucesso
</ResponseField>

<ResponseField name="failedMessages" type="number">
  Mensagens que falharam
</ResponseField>

<ResponseField name="createdAt" type="datetime">
  Data e hora de criação
</ResponseField>

<ResponseField name="updatedAt" type="datetime">
  Data e hora da última atualização
</ResponseField>

<ResponseField name="scheduledFor" type="datetime">
  Data e hora agendada (null se execução imediata)
</ResponseField>

<ResponseField name="evolutionUrl" type="string">
  URL da Evolution API usada
</ResponseField>

<ResponseField name="evolutionInstance" type="string">
  Instância do WhatsApp usada
</ResponseField>

<ResponseField name="message" type="string">
  Mensagem de erro (null em caso de sucesso)
</ResponseField>

## Códigos de Erro

| Código | Descrição |
|--------|-----------|
| 400 | Access token inválido ou campanha não encontrada |
| 500 | Erro interno do servidor |
