# Registrar Empresa

Registra uma nova empresa no sistema e retorna um access token para autenticação.

<Note>
  Este endpoint é usado apenas para registro inicial. Após obter o access token, use-o em todos os outros endpoints.
</Note>

## Endpoint

```http
POST /api/auth/register
```

## Headers

| Header | Tipo | Obrigatório | Descrição |
|--------|------|-------------|-----------|
| `Content-Type` | string | Sim | `application/json` |

## Request Body

<ParamField body="name" type="string" required={false}>
  Nome da empresa (opcional). Se não fornecido, será usado o nome do email.
</ParamField>

<ParamField body="email" type="string" required={true}>
  Email da empresa. Deve ser um email válido e único.
</ParamField>

<ParamField body="password" type="string" required={true}>
  Senha da empresa. Deve ter pelo menos 6 caracteres.
</ParamField>

## Exemplo de Request

```bash
curl -X POST https://api.disparador.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Minha Empresa LTDA",
    "email": "contato@minhaempresa.com",
    "password": "senha123"
  }'
```

## Response

### Sucesso (200 OK)

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjb21wYW55SWQiOjEsImlhdCI6MTcwNTQ0NzIwMCwiZXhwIjoxNzA1NTMzNjAwfQ.example",
  "companyId": 1,
  "message": null
}
```

### Erro (400 Bad Request)

```json
{
  "accessToken": null,
  "companyId": null,
  "message": "Email já está em uso"
}
```

## Campos da Resposta

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `accessToken` | string | Token de acesso para autenticação. Use no header `X-Access-Token` |
| `companyId` | number | ID único da empresa registrada |
| `message` | string | Mensagem de erro (null em caso de sucesso) |

## Códigos de Erro

| Código | Descrição |
|--------|-----------|
| `400` | Dados inválidos ou email já em uso |
| `500` | Erro interno do servidor |

## Validações

- **Email**: Deve ser um email válido e único no sistema
- **Senha**: Mínimo 6 caracteres
- **Nome**: Opcional, máximo 100 caracteres

## Exemplos de Erro

### Email já em uso
```json
{
  "accessToken": null,
  "companyId": null,
  "message": "Email já está em uso"
}
```

### Email inválido
```json
{
  "accessToken": null,
  "companyId": null,
  "message": "Email deve ter formato válido"
}
```

### Senha muito curta
```json
{
  "accessToken": null,
  "companyId": null,
  "message": "Senha deve ter pelo menos 6 caracteres"
}
```
