</ParamField>

<ParamField body="credentials" type="object" required>

## Autenticação

A API utiliza um sistema de autenticação baseado em tokens por empresa:

<CardGroup cols={3}>
  <Card title="Login" icon="sign-in" color="#16a34a">
    Faça login com email e senha da empresa
  </Card>
  <Card title="Token Header" icon="key" color="#eab308">
    Use o token no header X-Access-Token
  </Card>
  <Card title="Sem Expiração" icon="infinity" color="#6366f1">
    Tokens não expiram automaticamente
  </Card>
</CardGroup>

<CodeGroup>

```bash Login
curl -X POST https://api.exemplo.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "contato@empresa.com",
    "password": "senha123"
  }'
```

</CodeGroup>

<ResponseExample>

```json Resposta de Login
{
  "accessToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "tokenType": "Access",
  "companyId": 1,
  "companyName": "Minha Empresa",
  "companyEmail": "contato@empresa.com",
  "blockedContactsEnabled": true,
  "pollCampaignsEnabled": true,
  "expiresAt": null
}
```

</ResponseExample>

## Tipos de Mensagem Suportados

A API suporta diversos tipos de mensagem para atender diferentes necessidades:

| Tipo | Descrição | Campos Obrigatórios |
|------|-----------|-------------------|
| **TEXT** | Mensagem de texto simples | `message` |
| **IMAGE** | Imagem com legenda opcional | `mediaUrl`, `fileName`, `caption` |
| **VIDEO** | Vídeo com legenda opcional | `mediaUrl`, `fileName`, `caption` |
| **AUDIO** | Áudio/arquivo de voz | `mediaUrl` |
| **DOCUMENT** | Documentos (PDF, DOC, etc) | `mediaUrl`, `fileName` |
| **POLL** | Enquetes interativas | `pollData` |

## Configuração Rápida

Para desenvolvedores que desejam executar o sistema localmente:

<CodeGroup>

```bash Docker Compose
# Clone o repositório
git clone https://github.com/matheusluislorscheiter/api-disparador

# Configure as variáveis de ambiente
cp .env.example .env

# Execute com Docker Compose
docker-compose up -d
```

```bash Execução Manual
# Backend
cd backend
./mvnw spring-boot:run

# Frontend (em outro terminal)
cd frontend
npm install
npm run dev
```

</CodeGroup>

<Note>
- Backend: `http://localhost:8080`
- Frontend: `http://localhost:5173`
- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`
</Note>
